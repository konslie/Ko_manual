성과지표 산출 로직

2.1 확정 매출 산출 및 보정

고객 ID 단위로 집계된 확정 매출을 기준으로 합니다. 단, 사업자번호 없이 조직 간 보정 전표만 처리되는 일부 고객사는 원장에서 누락되므로 수기 보정이 필요합니다.

⚠️ 매출 누락 보정 대상 고객 — 월별 수동 기입 필수
누락 발생 원인: 사업팀 귀속 매출을 영업 귀속으로 보정하는 과정에서 사업자번호 없이 조직 간 전표 처리가 발생하여 고객단위 집계에서 누락됩니다.
아래 고객사들이 주요 보정 대상이며, 전체 합산 월평균 4~5억 원 규모입니다.
에스원 KGM 토요타
📌 관리 규칙: 매월 보정 매출 발생 시 '사업팀 귀속 → 영업 귀속' 보정액을 매출조정 수기테이블에 필수 수동 기입

특정 고객사 예외 처리 규칙

일반 로직을 그대로 적용하면 왜곡이 발생하는 고객사입니다. 반드시 숙지하세요.

네이버 코람코 가산센터 (IDC)
예외 #1
✗ 문제
심의 규모가 비대하고 이익률이 약 1.53%로 극히 낮으나, 초기 매출이 과소 계상되어 이익률 왜곡 발생
✓ 조정
심의상 매출 대신 실제 청구 매출(Billing) 기준으로 역산 적용
한전 (KEPCO) 전력통신
예외 #2
✗ 문제
당사 매출 비중이 매우 크나, 등록된 VRB 심의 규모가 극히 작아 일반 로직 적용 시 정합성 훼손
✓ 조정
심의 유무와 무관하게 전력통신 손익 전체에 PL 이익률 일괄 강제 적용

예외 규칙 요약 비교

고객사 일반 로직 왜곡 원인 적용 예외 규칙
네이버 코람코 (IDC) 심의 매출 기준 역산 심의 이익률 1.53% + 초기 매출 과소 Billing 매출 기준으로 역산 적용
한전 KEPCO 전력통신 VRB 이력 있으면 심의 이익률 적용 VRB 심의 규모 극소 → 대형 고객 대표성 없음 VRB 무관, PL 이익률 강제 일괄 적용
